Note a few things:

  • Guava automatically creates getters and setters for object attributes if you leave off the "access modifier" (i.e. the private, public, or protected), so even though in the above code it looks like we're just accessing the attributes directly, you could access them via getter/setter syntax as well if you wanted.
  • Note that since we didn't define a constructor for our Person object, we can instantiate it using the default method of using named arguments.
  • You Java folks will probably wonder what in the world that Comparator declaration is doing. Basically, Groovy has a shortcut where you can create a map which maps "method names" to Groovy closures, and then use the as keyword to cast it to an interface which has that method.
  • The @ToString annotation eliminates the need to override the Object.toString() method yourself.
